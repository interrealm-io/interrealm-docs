InterRealm Nexus Console Walkthrough

The Nexus Console is the open source reference implementation of InterRealm — a protocol for recursive, distributed AI collaboration.

1. Realm Hierarchy

Realms are universal boundaries for intelligence. They can represent organizations, systems, or logical domains. Nexus supports nested realms with full visibility and control.

2. Realm Overview

Each realm has a type, parent, members, and policies. Realms are the core unit of distributed intelligence in InterRealm.

3. Members: Agents, Apps, and Bridges

Members are entities within a realm. They can be AI agents, external applications (with API keys), or bridges to other realms across the internet.

4. Application Members & API Keys

Application members receive unique API keys and IDs, enabling secure access to realm capabilities and recursive invocation.

5. Capabilities Schema

Capabilities define what a member can do. They’re described in a schema that agents can discover and invoke dynamically.

6. Recursive Call Flow

Agents can call other agents recursively across realms, forming dynamic call stacks to solve complex problems collaboratively.

7. Bridge Types: Cross-Realm Connectivity

Bridges connect realms across the internet, enabling mesh routing, policy enforcement, and capability discovery across organizational boundaries.

8. Agent SDKs: Realm-Agnostic Intelligence

Agents don’t need to know they’re in a realm. SDKs abstract the protocol, allowing seamless registration, discovery, and invocation.
